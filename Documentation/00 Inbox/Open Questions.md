# Open Questions

## Brand

- What should the editor be called?
- Do we want distinct names for editor, player, and toolchain, or keep them under the RawIron family first?

## Engine

- Is Vulkan the first shipping backend, or only the first development backend?
- Does JavaScript become a formal embedded scripting layer, or only a tooling language?
- What is the first supported platform target after Windows?

## Assets

- Which internal extensions become final?
- Do editable and cooked assets use different extension families?
- What is the first canonical model/material schema?

## Workflow

- What does the first "RawIron editor loop" look like?
- Which prototype workflows are non-negotiable and must survive the rebuild?
