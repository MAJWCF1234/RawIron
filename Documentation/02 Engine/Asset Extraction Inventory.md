# Asset Extraction Inventory / Manifest

## Overview

The Asset Extraction Inventory / Manifest is a pipeline-side bookkeeping artifact that records the current state of archived source assets and their extracted contents. Its purpose is to provide Raw Iron tools and developers with a single structured view of what archives exist, what has been unpacked, and whether the extracted output is current relative to its source inputs.

This artifact is generated during asset preparation and is intended for development, import, and pipeline validation workflows rather than runtime scene execution.

## Purpose

Raw Iron is expected to support a mixed asset pipeline that may include archived source packages, unpacked intermediate content, and converted runtime-ready resources. In this context, the engine needs a reliable way to track what source bundles were discovered, what was extracted from them, and whether the extracted results remain valid.

The Asset Extraction Inventory exists to solve that tracking problem. It acts as a machine-readable manifest for asset extraction state, allowing tooling to reason about archive contents without rescanning the filesystem from scratch each time.

## Core Function

The manifest is generated by an extraction or asset-preparation tool and typically records information such as:

- generation timestamp
- discovered archive files
- archive paths and identifiers
- archive hash or signature information
- extraction status
- cache validity state
- unpacked output entries
- unpacked file paths, sizes, and modification times

This gives both developers and automated tooling a single source of truth for the extraction stage of the pipeline.

## Role in Raw Iron

Within Raw Iron, this artifact should be treated as a pipeline inventory record rather than a gameplay or rendering asset. It belongs to the asset tooling layer and exists to support:

- archive extraction workflows
- import validation
- cache invalidation logic
- tool-driven asset discovery
- reproducible content preparation

It should not be treated as part of the game loop, renderer, simulation state, or scene format.

## What It Is Not

This artifact is not:

- a runtime asset database
- a scene description
- a model format
- an in-engine manifest for gameplay use
- a serialization format for shipping builds

It is strictly a development and pipeline bookkeeping structure.

## Benefits

The Asset Extraction Inventory provides several practical benefits:

### Pipeline Visibility

It gives developers immediate visibility into what source archives were found and what files were unpacked from them.

### Cache Validation

It allows tools to determine whether extracted results are stale, current, or missing without relying solely on manual inspection.

### Automation Support

It gives import and validation scripts a stable structured input for downstream tasks such as conversion, indexing, or rebuild decisions.

### Reduced Pipeline Ambiguity

It helps distinguish between “no assets were found,” “assets were found but not extracted,” and “assets were extracted successfully but are now outdated.”

## Summary

The Asset Extraction Inventory / Manifest is a generated pipeline artifact that records the discovery and extraction state of archived source assets. Its role is to support Raw Iron’s asset preparation workflow by making extraction results visible, structured, and machine-verifiable without tying that bookkeeping to the runtime itself.

## Native Implementation

Native types and JSON helpers live in `RawIron.Content`:

- Header: `Source/RawIron.Content/include/RawIron/Content/Pipeline/AssetExtractionInventory.h`
- Namespace: `ri::content::pipeline`

Serialize/parse/load/save APIs record `formatVersion`, `generatedAtUtc`, and per-archive entries (`path`, `identifier`, `signature`, `extractionStatus`, `extractedOutputCurrent`, `extractedOutputs` with `relativePath`, `sizeBytes`, `modifiedUtc`).
