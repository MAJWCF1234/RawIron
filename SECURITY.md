# Security

## Supported versions

Security-sensitive fixes are applied on the default branch (**`main`**) first. There is no separate long-term support line today; use the latest **`main`** or the most recent **GitHub Release** asset you trust.

## Reporting a vulnerability

**Please do not** file a public issue for undisclosed security problems in this repository.

Instead, use **GitHub’s private vulnerability reporting** for this repo ( **Security** tab → **Report a vulnerability** ), if enabled by the maintainers. If that option is unavailable, contact the repository owners through a **private** channel they publish in the repo **README** or organization profile.

Include:

- A short description of the issue and its impact  
- Steps to reproduce (build target, command line, minimal asset or script if needed)  
- Affected commit or release tag, if known  

We will treat reports as confidential and coordinate a fix and disclosure timeline when possible.
